## What is dio?

A powerful HTTP client for Dart and Flutter, which supports global settings, Interceptors, FormData, aborting and canceling a request, files uploading and downl. It focuses on talking to APIs, databases, and persistence layers. The project lives at [cfug/dio](https://github.com/cfug/dio) and is maintained by `cfug`.

## Installing dio

Add the package to your `pubspec.yaml`:

```yaml
dependencies:
  dio: ^latest
```

Then fetch it and import it in your Dart code:

```bash
flutter pub get
```
```dart
import 'package:dio/dio.dart';
```

Check the package's `example/` directory and its [pub.dev page](https://pub.dev/packages/dio) for the exact API — dio is versioned there with full docs so you always integrate against the current release.

## When should you use dio?

Reach for dio when:

- you're calling REST/GraphQL APIs from a Flutter app
- you need local storage, caching, or offline sync
- you want typed, testable data access
