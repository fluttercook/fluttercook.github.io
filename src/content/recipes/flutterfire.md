## What is flutterfire?

A collection of Firebase plugins for Flutter apps. It focuses on talking to APIs, databases, and persistence layers. The project lives at [firebase/flutterfire](https://github.com/firebase/flutterfire) and is maintained by `firebase`.

## Installing flutterfire

Add the package to your `pubspec.yaml`:

```yaml
dependencies:
  flutterfire: ^latest
```

Then fetch it and import it in your Dart code:

```bash
flutter pub get
```
```dart
import 'package:flutterfire/flutterfire.dart';
```

Check the package's `example/` directory and its [GitHub repo](https://github.com/firebase/flutterfire) for the exact API — flutterfire is versioned there with full docs so you always integrate against the current release.

## When should you use flutterfire?

Reach for flutterfire when:

- you're calling REST/GraphQL APIs from a Flutter app
- you need local storage, caching, or offline sync
- you want typed, testable data access

It's especially relevant if your project touches `dart`, `firebase`, `flutter`, `google`.
